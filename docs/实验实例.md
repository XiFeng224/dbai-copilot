## 实验实例设计

### 实验实例1：实时性能监控

**场景**：模拟数据库高并发场景，监控系统性能变化

**步骤**：
1. **准备环境**：
   - 启动MySQL数据库
   - 导入测试数据（如100万条订单数据）
2. **配置连接**：
   - 在数据库运维Agent中配置数据库连接信息
   - 点击"连接数据库"按钮
3. **启动监控**：
   - 点击"采集指标"按钮，记录初始性能指标
4. **模拟负载**：
   - 使用sysbench或JMeter模拟100并发的查询请求
   - 执行以下SQL语句：
     ```sql
     SELECT * FROM orders WHERE status='pending' ORDER BY created_at DESC LIMIT 100;
     ```
5. **观察指标**：
   - 实时查看QPS、TPS、CPU使用率、内存使用率等指标变化
   - 记录峰值和平均值

**预期结果**：
- QPS值会随着并发请求增加而上升
- CPU使用率会相应增加
- 内存使用率保持稳定
- 系统能够实时捕获性能变化

### 实验实例2：SQL执行计划分析

**场景**：分析一个性能较差的SQL语句，识别瓶颈并优化

**步骤**：
1. **准备数据**：
   - 确保orders表有大量数据（100万条）
   - 确保status字段没有索引
2. **输入SQL**：
   - 在"诊断分析"标签页输入：
     ```sql
     SELECT * FROM orders WHERE status='pending' ORDER BY created_at DESC;
     ```
3. **分析执行计划**：
   - 点击"分析执行计划"按钮
   - 查看执行计划的DAG图
4. **查看优化建议**：
   - 系统会提示"全表扫描"和"文件排序"问题
   - 推荐添加索引
5. **验证优化**：
   - 添加索引：`CREATE INDEX idx_status_created_at ON orders(status, created_at);`
   - 重新分析执行计划
   - 比较优化前后的执行时间

**预期结果**：
- 优化前：执行计划显示"ALL"（全表扫描）和"Using filesort"
- 优化后：执行计划显示"range"（索引扫描），无文件排序
- 执行时间显著减少

### 实验实例3：索引推荐

**场景**：系统自动分析SQL语句并推荐合适的索引

**步骤**：
1. **输入复杂SQL**：
   - 在"智能优化"标签页输入：
     ```sql
     SELECT o.id, o.customer_id, c.name 
     FROM orders o 
     JOIN customers c ON o.customer_id = c.id 
     WHERE o.status='completed' AND o.amount > 1000 
     ORDER BY o.created_at DESC;
     ```
2. **获取推荐**：
   - 点击"推荐索引"按钮
3. **查看建议**：
   - 系统会推荐在orders表的status、amount和created_at字段上创建索引
   - 推荐在customers表的id字段上创建主键索引（如果不存在）
4. **验证效果**：
   - 创建推荐的索引
   - 执行SQL语句，比较执行时间

**预期结果**：
- 系统能够准确识别SQL语句中的过滤条件和排序字段
- 推荐的索引能够显著提高查询性能
- 执行时间减少50%以上

### 实验实例4：智能诊断报告

**场景**：生成数据库健康状况的智能诊断报告

**步骤**：
1. **准备环境**：
   - 确保数据库运行一段时间，积累足够的性能数据
   - 确保有一些慢查询记录
2. **生成报告**：
   - 在"AI对话"标签页点击"生成诊断报告"按钮
3. **分析报告**：
   - 查看报告中的性能概览、慢查询分析、锁分析和参数调优建议
4. **实施建议**：
   - 根据报告中的建议优化数据库配置
   - 重新生成报告，比较优化前后的变化

**预期结果**：
- 报告能够全面分析数据库的健康状况
- 提供具体的优化建议
- 优化后数据库性能明显提升

### 实验实例5：自然语言问答

**场景**：通过自然语言提问，获取数据库相关信息

**步骤**：
1. **输入问题**：
   - 在"AI对话"标签页输入：
     "我的数据库性能最近下降了，可能是什么原因？"
     "如何优化我的慢查询？"
     "我的数据库连接数过多，怎么办？"
2. **获取回答**：
   - 点击"回答问题"按钮
3. **验证回答**：
   - 检查回答的准确性和实用性
   - 根据回答中的建议进行操作，验证效果

**预期结果**：
- 系统能够理解自然语言问题
- 提供准确、专业的回答
- 回答中的建议能够有效解决实际问题
